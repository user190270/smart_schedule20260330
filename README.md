# Smart Schedule

一个面向移动端场景的本地优先日程系统，集成 AI 解析、知识库问答与分享能力。核心流程仍由应用状态机、同步策略和明确的用户确认驱动，而不是把整个产品建立成纯 LLM 驱动的 AI Native 形态。

## 项目包含什么

- `Vue 3 + TypeScript` 前端
- `FastAPI` 后端
- `PostgreSQL + pgvector` 云端数据链路
- 本地优先日程仓与显式 `Push / Pull`
- 面向日程场景的解析 Agent，在既有产品流程中承担多轮澄清、草稿维护与确认前辅助生成
- 知识库重建、RAG 问答与分享链路
- `Capacitor Android` 移动端壳，用于真机或模拟器验证

## 当前产品方向

- 日程的创建、编辑、删除首先发生在本地仓，而不是直接把云端当主视图。
- 云端同步、知识库重建都是显式动作，不会被“刷新列表”伪装掉。
- AI 解析只负责理解自然语言、追问缺失字段并维护草稿；真正保存仍由应用在用户确认后执行。
- 前端只调用本项目自己的后端，不会把第三方模型 API 直接暴露给客户端。

## 仓库说明

- 本仓库保留项目主代码与产品文档，便于展示和部署协作。
